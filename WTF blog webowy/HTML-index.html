HTML 1.01

Czym jest HTML?

HTML to skrót od Hypertext Markup Language. Jest to nic innego jak jezyk opisowy oparty na tzw. znacznikach (tagach) opisujacy zawartość strony internetowej.
SEMANTYCZNY HTML oznacza precyzyjne definiowanie przeznaczenia poszczególnych części kodu źródłowego.
*semantyczny (HTML) – mający znaczenie i stąd opierający sie na znacznikach;

Znacznik (tag) to element składowy dokumentu HTML, który jest zapisywany z użyciem nawiasów ostrych;
I tak wyróżniamy:
- znacznik pojedynczy < img />
- znacznik podwójny < a> < /a>

Atrybut to cecha (właściwość) znacznika, zdefiniowana w jego wnętrzu, w postaci: nazwa_atrybutu=”wartość”;

Znacznik zbudowany jest w następujący sposób: 

<znacznik atrybut=”wartość”>treść </znacznik>

Znacznik definiujący hiperłącze (link) to < a> < /a>natomiast href to adres strony (od hyper text reference), do której ma prowadzić link;i tak:<a href=”adres-strony.pl”>LINK</a> A więc przykładowo jeżeli chcemy umieścić link do strony .pl to zapiszemy:
< a href=”http://onet.pl” target=”_blank” title=”hint” >KLIKNIJ</a>
*na ekranie widać odnośnik KLIKNIJ, który jest linkiem do strony onet.pl;

Koniecznie należy pamiętać o umieszczeniu pełnego adresu strony, włącznie z protokołem http://chyba, że adres mieści się w Twojej własnej domenie; target=_blankspowoduje, iż łącze otworzy się na nowej stronie; natomiast znacznik title=”hint” utworzy napis przy kursorze po najechaniu na link, tzw. hint;
Reasumując...HTML to język opisany za pomocą znaczników. Wyróżniamy znaczniki pojedyncze i podwójne. Znacznikom przypisywane są atrybuty, a tym konkretne wartości.

HTML 1.02

Formatowanie tekstu w kodzie. Odnośniki i znaczniki w HTML.

Jeżeli chcemy umieścić treść na naszej stronie internetowej czy blogu musimy ten tekst sformatować (w HTML-u) i dalej wystylizować (w CSS-ie). Tekst, który wprowadzimy do kodu HTML nie będzie wyglądał tak samo na naszej stronie. Do tego używamy znaczników. Jeżeli chcemy rozpocząć zdanie od nowej linijki należy użyć znacznika < /br> . Jeżeli chcemy wybrany fragment tekstu pogrubić użyjemy < b> < /b> , a więc bold. Jeżeli chcemy wprowadzić tekst pochylony użyjemy < i> < /i> co znaczy italic. Identyczne dla oka funkcje spełniają znaczniki odpowiednio < strong> < /strong> oraz < em> < /em> . Różnica polega na tym, że te drugie mają również znaczenie semantyczne dla czytników stron. Jezeli chcemy podkreslić dany fragment tekstu użyjemy znacznika < u> co znaczy underline.

Odnosniki: poczta i podstrona. 

Bardzo często, przeważnie w stopce strony, umieszczamy pole kontaktu z autorem czy to bloga czy strony internetowej. Jak będzie wyglądał taki odnośnik? I tu drobna uwaga. Nie ma sie co łudzić, że adres zapisany na zasadzie imie.nazwisko_at_nazwadomeny.pl bedzie niewidzialny dla robotów. One pobierają adres z kodu HTML. Z drugiej strony przeróżne instytucje, jak np. banki poprostu sprzedają nasze adresy poczty.

    Odnośniki:
  • o adresie bezwzględnym:
    < a href="https://www.google.pl">Przejdź do googla< /a>
  • o adresie względnym:
    < a href="galeria.html">Przejdż do galerii
  • odnośnik do poczty:
    Napisz do mnie: < a href="mailto:imie.nazwisko@gmail.com">imie.nazwisko@gmail.com< /a>

    • Nie należy umieszczać na stronie odnośnika pt. Napisz do mnie w miejsce adresu poczty. Będzie on poprostu przekierowywał do nieskonfigurowej przeglądarki poczty elektronicznej.

HTML 1.03

Listy i tabele w HTML.

Rozróżniamy dwa rodzaje list: ordered list < ol> < /ol> czyli listy numerowane/uszeregowane w kolejności oraz undordered list < ul> < /ul> czyli listy nienumerowane.
Zapis listy nienumerowanej wygląda następująco:

    < ul>
  •   < li> Punkt 1 < /li>
  •   < li> Punkt 2 < /li>
  •   < li> Punkt 3 < /li>
  •   < li> Punkt 4 < /li>
  •   < li> Punkt 5 < /li>
    • < /ul>
Na stronie zobaczymy listę w kształcie:
  • Punkt 1
  • Punkt 2
  • Punkt 3
  • Punkt 4
  • Punkt 5
Zapis listy numerowanej będzie wyglądał analogicznie. Zmienimy jedynie znacznik < ul> < /ul> na < ol> < /ol>:
    < ol>
  1.   < li> Punkt 1 < /li>
  2.   < li> Punkt 2 < /li>
  3.   < li> Punkt 3 < /li>
  4.   < li> Punkt 4 < /li>
  5.   < li> Punkt 5 < /li>
    6. < /ol>
A w rezultacie:
  1. Punkt 1
  2. Punkt 2
  3. Punkt 3
  4. Punkt 4
  5. Punkt 5

Niekiedy listy zagnieżdżamy, tzn umieszczamy listy wew list, tzn:

    < ul>
  •   < li> Punkt 1 < /li>
      < ol>
    1.   < li> Punkt 1 < /li>
    2.   < li> Punkt 2 < /li>
    3.   < li> Punkt 3 < /li>
      4. < /ol>
  •   < li> Punkt 2 < /li>
  •   < li> Punkt 3 < /li>
  •   < li> Punkt 4 < /li>
  •   < li> Punkt 5 < /li>
    • < /ul>
W efekcie otrzymujemy:
  • Punkt 1
    1. Podpunkt 1
    2. Podpunkt 2
    3. Podpunkt 3
  • Punkt 2
  • Punkt 3
  • Punkt 4
  • Punkt 5
